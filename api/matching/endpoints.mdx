## Overview

The Matching API provides intelligent recommendations by analyzing user profiles, preferences, languages, expertise areas, and locations to suggest the best matches between tourists and guides.

## Get Recommended Guides for Tourist

<CodeGroup>
```bash cURL
curl -X GET "https://api.kinconecta.com/api/matching/tourist/123/guides?limit=10&offset=0" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const response = await fetch(
  'https://api.kinconecta.com/api/matching/tourist/123/guides?limit=10&offset=0',
  {
    method: 'GET',
    headers: {
      'Authorization': 'Bearer YOUR_TOKEN'
    }
  }
);
const data = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://api.kinconecta.com/api/matching/tourist/123/guides',
    params={'limit': 10, 'offset': 0},
    headers={'Authorization': 'Bearer YOUR_TOKEN'}
)
data = response.json()
```
</CodeGroup>

### Endpoint

```
GET /api/matching/tourist/{touristUserId}/guides
```

### Path Parameters

<ParamField path="touristUserId" type="Long" required>
  The unique identifier of the tourist user requesting guide recommendations
</ParamField>

### Query Parameters

<ParamField query="limit" type="Integer" default="20">
  Maximum number of recommended guides to return (pagination)
</ParamField>

<ParamField query="offset" type="Integer" default="0">
  Number of results to skip for pagination
</ParamField>

### Response

<ResponseField name="results" type="Array<MatchResultDTO>">
  List of recommended guide matches with scoring details
  
  <Expandable title="MatchResultDTO properties">
    <ResponseField name="candidateUserId" type="Long">
      The user ID of the recommended guide
    </ResponseField>
    
    <ResponseField name="score" type="double">
      Matching algorithm score (0-100) indicating compatibility
    </ResponseField>
    
    <ResponseField name="matchedFields" type="Array<MatchedFieldDTO>">
      Detailed breakdown of matched criteria
      
      <Expandable title="MatchedFieldDTO properties">
        <ResponseField name="field" type="String">
          Name of the matched field (e.g., "language", "interest", "location")
        </ResponseField>
        
        <ResponseField name="evidence" type="String">
          Specific value that matched (e.g., "Spanish", "Hiking", "Madrid")
        </ResponseField>
        
        <ResponseField name="points" type="double">
          Points contributed to the overall match score
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="profilePreview" type="ProfilePreviewDTO">
      Quick profile summary of the guide
      
      <Expandable title="ProfilePreviewDTO properties">
        <ResponseField name="fullName" type="String">
          Guide's full name
        </ResponseField>
        
        <ResponseField name="avatarUrl" type="String">
          URL to the guide's profile picture
        </ResponseField>
        
        <ResponseField name="coverUrl" type="String">
          URL to the guide's cover photo
        </ResponseField>
        
        <ResponseField name="locationLabel" type="String">
          Primary location where the guide operates
        </ResponseField>
        
        <ResponseField name="rating" type="BigDecimal">
          Average rating from previous reviews
        </ResponseField>
        
        <ResponseField name="reviewsCount" type="Integer">
          Total number of reviews received
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="totalCandidates" type="long">
  Total number of potential guide matches available
</ResponseField>

<ResponseField name="limit" type="int">
  The limit value used in this query
</ResponseField>

<ResponseField name="offset" type="int">
  The offset value used in this query
</ResponseField>

## Matching Algorithm

The matching algorithm analyzes multiple factors to calculate compatibility scores:

### Scoring Factors

- **Languages** (up to 25 points): Matches between tourist language preferences and guide spoken languages
- **Interests & Expertise** (up to 30 points): Alignment between tourist interests and guide expertise areas
- **Location** (up to 30 points): Geographic proximity and location preferences
- **Adaptations** (up to 15 points): Special accessibility needs and guide capabilities

### Score Interpretation

- **90-100**: Excellent match - Highly recommended
- **75-89**: Very good match - Strong compatibility
- **60-74**: Good match - Solid potential
- **Below 60**: Moderate match - Consider other options
